# 课程网站质量审阅报告

## 一、总体判断

当前网站已经具备学生端课程资料站的基础结构，能够通过 `mkdocs build --strict` 构建，也未发现考试保密资料进入公开目录。但从学生使用体验看，当前版本仍更接近“资料下载目录”，还没有形成清晰的课程学习平台。

核心问题集中在三类：

1. 核心资料在线阅读能力不足：电子教材和 PPT 只提供下载链接，学生需要离开网页打开 PDF。
2. 作业与项目材料呈现不够友好：作业正文虽然在页面中出现，但仍链接到 `.txt` 附件，容易出现浏览器编码识别错误。
3. 首页和课程页面视觉层级偏弱：缺少 Hero 区、入口卡片、学习路径和资源标签，课程网站质感不足。

## 二、逐项审阅

| 检查项 | 当前状态 | 问题判断 |
|---|---|---|
| 首页是否像课程门户 | 部分满足 | 只有文字、表格和学习主线，缺少 Hero 区和入口卡片 |
| 电子教材入口 | 有 | 只提供下载链接，缺少在线预览 |
| 每次课 PPT | 有 | 只提供下载链接，缺少在线预览 |
| 课后作业 | 有 | 页面正文有作业，但仍提供 `.txt` 下载，存在乱码风险 |
| 课程项目 M1-M4 | 有 | 结构清楚，但视觉层级和模板呈现可继续增强 |
| 补充资料 | 有 | 内容偏薄，分类不足，像说明文字而不是资源中心 |
| 复习说明 | 有 | 内容安全，未泄露试题，但页面还可以更结构化 |
| 文件编码 | 基本可控 | `.txt` 文件为潜在风险，应统一 UTF-8 并减少直接点击 `.txt` |
| 视觉质感 | 一般 | MkDocs 默认样式为主，缺少课程专属 CSS |
| 保密风险 | 当前通过 | 未发现 `exam/`、teacher_only、answer_key、paper_A/paper_B 等文件进入公开目录 |

## 三、主要问题

### 1. 首页不像学习门户

当前首页有课程介绍和快速入口表格，但缺少：

- 大标题 Hero 区；
- 四个核心入口卡片；
- 学习路径图；
- 最新/重点资源入口；
- 学生使用指引。

这会让学生进入网站后仍需要“找文件”，而不是按学习路径进入课程。

### 2. PDF/PPT 没有在线预览

当前电子教材和每次课 PPT 均以下载链接形式出现。对学生来说，尤其在手机或浏览器中使用时，下载 PDF 后再打开会增加使用成本。

建议改为：

- 页面内嵌 PDF 预览；
- 保留“下载 PDF”按钮；
- PPTX 不直接公开，继续以 PDF 为主。

### 3. 作业 `.txt` 存在乱码与体验问题

当前每次课页面有“下载课后作业文本”链接，目标是 `.txt` 文件。即使文件实际是 UTF-8，浏览器仍可能按本地默认编码或 `text/plain` 无 charset 处理，导致乱码。

建议：

- 作业要求直接转为页面正文；
- 保留 Markdown 附件或不提供附件；
- 不再让学生打开 `.txt` 查看核心作业。

### 4. 课程项目页面可读性还可以增强

M1-M4 页面已经有提交材料和格式要求，但更适合加入：

- 阶段标签；
- 提交材料卡片；
- 建议模板按钮；
- 评价关注点。

注意：评价关注点只能是学生可见的质量要求，不应公开教师评分细则。

### 5. 补充资料页面偏薄

当前补充资料只有三个小节，未形成分类资源中心。建议按主题分为：

- AI 时代软件工程；
- 架构决策与 ADR；
- 测试与质量保障；
- 发布运维与可观测性；
- 推荐使用方式。

### 6. 保密边界需要继续自动化检查

当前泄密检查通过，但后续优化会新增页面和资源，必须继续运行脚本检查：

- `teacher_only`
- `answer_key`
- `paper_A`
- `paper_B`
- `question_bank`
- `exam_quality_review`
- `答案`
- `评分细则`
- `教师版`
- `命题质量分析`

## 四、修复优先级

| 优先级 | 问题 | 修复方式 |
|---|---|---|
| P0 | 考试资料泄露风险 | 保留并增强自动扫描 |
| P1 | PDF/PPT 只能下载 | 为教材和每次课 PPT 增加 iframe 在线预览 |
| P1 | `.txt` 作业乱码风险 | 统一编码，作业正文转 Markdown 展示 |
| P2 | 首页像目录 | 增加 Hero、入口卡片、学习路径 |
| P2 | 每次课页面单薄 | 统一课次页面模块和资源标签 |
| P2 | 项目页面平淡 | 增加阶段卡片、模板入口和质量关注点 |
| P3 | 补充资料偏薄 | 分类重构资源说明 |

## 五、结论

当前网站适合作为第一版资料站，但尚未达到“课程学习平台”的体验。建议按本报告进入结构优化：保留 MkDocs Material，新增 CSS、PDF 预览、作业正文化、资源卡片和自动化检查脚本。
