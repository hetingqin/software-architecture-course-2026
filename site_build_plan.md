# 《软件体系结构》学生端课程网站建设方案

## 一、建设定位

本网站面向选课学生，用于集中发布《软件体系结构》课程的公开学习资料。网站不是考试系统、作业提交系统，也不是教师命题资料库。

学生端网站只发布以下四类资料：

1. 电子教材学生阅读版；
2. 每次课 PPT PDF；
3. 课后作业与课程项目里程碑要求；
4. 额外知识资料、扩展阅读和不泄露试题的复习范围说明。

所有考试保密资料不得进入 `site/docs/`、`site/public/` 或任何可被学生访问的站点目录。

## 二、建设原则

| 原则 | 说明 |
|---|---|
| 公开资料白名单 | 只复制明确允许公开的文件，不采用“整个目录批量复制” |
| 考试资料默认保密 | `exam/` 目录整体默认不进入学生网站 |
| PDF 优先发布 | PPT 优先发布 PDF，不直接发布可编辑 PPTX，避免误带备注或未清理内容 |
| Markdown 管理 | 页面正文用 Markdown 维护，便于后续更新 |
| 静态部署 | 不建设登录、提交、在线测验等复杂功能 |
| 可追溯更新 | 每次发布前保留公开清单和排除清单 |

## 三、建议发布范围

### 3.1 电子教材

建议公开发布：

- `textbook/output/software_architecture_textbook_student_v5_1.pdf`

不建议默认公开：

- `textbook/output/software_architecture_textbook_student_v5_1.docx`
- 教师版教材；
- 教材 source 版本；
- 中间审阅报告与生成报告。

DOCX 可作为教师维护源，不建议作为学生端主入口，避免版本混乱。

### 3.2 每次课资料

建议每次课页面统一包含：

- 主题名称；
- 本讲学习目标；
- PPT PDF 下载；
- 课后作业或项目里程碑要求；
- 关键词；
- 与课程项目的关系；
- 可选扩展阅读。

PPTX、逐字讲稿、教师备注、生成脚本、审阅报告默认不公开。

### 3.3 课程项目资料

建议公开发布：

- M1：选题与需求分析；
- M2：建模与结构表达；
- M3：架构决策与 ADR；
- M4：测试、交付与运维说明；
- 可填写模板、检查清单、提交说明。

平时成绩教师评分细则不公开。学生端只发布“提交什么、如何准备、质量要求是什么”。

### 3.4 复习说明

可以公开发布一份“期末复习范围与复习方向说明”，但必须重新编写，不能直接复制：

- A/B 卷；
- 教师题库；
- 答案与评分细则；
- 命题质量分析；
- `exam_scope_note_teacher_only.md`。

公开复习说明只能说明章节范围、能力要求和复习建议，不能泄露正式试题、答案或评分点。

## 四、禁止发布范围

以下内容不得进入学生网站：

1. `exam/` 目录中的全部命题材料；
2. A/B 卷学生作答版；
3. 答案与评分细则；
4. 教师题库；
5. 命题质量分析报告；
6. 平时成绩教师评分方案；
7. 任何文件名包含 `teacher_only`、`教师版`、`答案`、`评分`、`试卷`、`题库`、`命题` 的材料；
8. 逐字讲稿、教师备注、PPT 生成中间稿；
9. debug、review、source、scripts 中的内部工程文件。

## 五、建议站点资产目录

后续确认后，可按以下结构整理公开资源：

```text
site/
├─ docs/
│  ├─ index.md
│  ├─ course/
│  ├─ textbook/
│  ├─ lessons/
│  ├─ project/
│  ├─ resources/
│  ├─ review/
│  └─ assets/
│     ├─ textbook/
│     ├─ lessons/
│     │  ├─ lesson01/
│     │  ├─ lesson02/
│     │  └─ ...
│     ├─ project/
│     └─ resources/
└─ scripts/
   └─ validate_public_assets.py
```

说明：当前阶段只规划，不复制文件。

## 六、建设流程

### 阶段1：发布审查

输出公开资产清单、保密资产清单、网站结构和技术选型。当前任务属于本阶段。

### 阶段2：白名单复制

确认后，只按 `public_asset_inventory.md` 中列出的文件复制到 `site/docs/assets/`。

### 阶段3：页面生成

基于 Markdown 生成：

- 首页；
- 课程说明；
- 电子教材；
- 每次课页面；
- 课程项目；
- 补充资料；
- 复习说明。

### 阶段4：保密校验

构建前运行校验脚本，检查 `site/docs/` 和 `site/public/` 是否包含禁用文件名或敏感内容。

建议禁止规则包括：

```text
exam
teacher_only
教师版
答案
评分
试卷
题库
命题
paper_A
paper_B
answer_key
debug
source
```

### 阶段5：部署

优先支持：

- GitHub Pages；
- Gitee Pages；
- 学校服务器静态目录；
- 校内对象存储或文件服务器。

## 七、发布前检查清单

| 检查项 | 要求 |
|---|---|
| 是否含考试资料 | 不得包含 |
| 是否含教师版资料 | 不得包含 |
| 是否含答案或评分细则 | 不得包含 |
| 是否含 PPTX 源文件 | 默认不包含 |
| PPT 是否为 PDF | 优先 PDF |
| 教材是否为学生版 v5.1 | 是 |
| 每次课是否有作业说明 | 优先有 |
| 复习说明是否不泄露试题 | 必须确认 |
| 链接是否可打开 | 构建后检查 |
| 文件名是否清晰 | 使用 lesson01、lesson02 等稳定命名 |
